# Edge SCADA / MES 扩展计划

更新时间：2026-06-17

## 1. 扩展定位

本系统后续定位为轻量级 Edge SCADA + 产线级 MES 节点。树莓派负责靠近设备侧的数据采集、短期存储、追溯、参数管控、报警、权限、归档调度和本地展示；高频原始采集、长期大数据分析、AI 重计算、图片/视频长期存储由外部设备或服务器承担。

目标规模：

- 最多支持 3 条流水线。
- 每条流水线可以包含 PLC、多个工站、参数清单、追溯、质量、报警、停机、OEE 与本地 dashboard。
- 系统必须离线可运行，网络恢复后再同步到服务器。

## 2. 明确纳入范围

### 2.1 多产线

- 系统目标规模为最多 3 条流水线。
- 每条线独立维护 `line_id`、PLC、工站、参数、工件、报警、质量和归档策略。
- 树莓派可以作为一个现场 Edge 节点管理多条线，但不承担集团级多工厂 MES 职责。

### 2.2 MCU 高频采集文件接入

高频信号采集由 MCU 或专用采集器完成，例如 STM32、ESP32 或其他边缘采集模块。Edge 系统不直接采样高频信号。

约定：

- MCU 输出文件格式：CSV 或 JSON。
- 文件粒度：每个零件生成一个文件。
- Edge 负责抓取、接收、解析、处理、分析和存档。
- 文件必须能够关联到零件或工站，优先使用 `unit_id`，也可以使用 DMC、station_id、时间戳作为辅助字段。
- 原始文件保存在文件系统或归档介质中，PostgreSQL 保存元数据、解析状态和特征值。

建议提取的特征：

- 最大值、最小值、平均值、标准差。
- 峰值时间、持续时间、采样点数量。
- 超限点数量、异常区间数量。
- 业务相关特征，例如扭矩峰值、电流峰值、压力积分、振动 RMS。

### 2.3 参数管理

参数管理是必要模块。Edge 负责集中读取、校验和审计，不主动回写 PLC。

约定：

- 参数修改发生在 PLC/HMI 或受控工程工具侧。
- PLC 侧参数修改后通知 Edge。
- Edge 收到通知后读取最新参数。
- Edge 根据参数规范进行校验。
- 被授权账号登录后即可修改参数，不需要审批流。
- 系统必须精确记录 changelog。

参数 changelog 至少包含：

- 参数 ID、参数名、参数路径。
- line_id、plc_id、station_id。
- 旧值、新值、单位、数据类型。
- 修改账号、认证方式、修改来源。
- PLC 通知时间、Edge 读取时间、Edge 记录时间。
- 校验结果、是否超出允许范围。

### 2.4 图片 / 视频接入

图片和视频来源为工业相机。Edge 负责接收元数据和短期本地缓存，不承担长期媒体库职责。

约定：

- 本地默认保留 7 天，可配置。
- 超期后按策略归档或删除。
- 文件与 `unit_id`、station_id、camera_id、capture_time、inspection_result 关联。
- PostgreSQL 保存元数据和索引，不保存大型媒体二进制。

### 2.5 归档与冷热备份

系统需要支持灵活归档策略。数据归档属于管理策略，应支持按现场规则调整。

归档周期：

- 支持按月或按季度强制归档。
- 后续可以扩展为按天、按周或手动任务。

归档目标：

- 冷备份：移动硬盘拷贝。
- 热备份：上传服务器。
- 两种目标都应作为可勾选选项。

归档对象：

- 数据库历史数据。
- MCU 原始文件。
- 工业相机图片/视频。
- 系统日志和采集日志。

### 2.6 权限与认证

权限管理需要纳入系统范围。

认证方式：

- 用户名/密码。
- 令牌验证。
- 生物识别外设，例如人脸识别模块、指纹模块。

约定：

- 生物识别作为外设认证适配器接入，不直接绑定业务模块。
- 参数修改、归档任务、配置变更等关键动作必须记录操作审计。
- 被授权用户登录后可执行其角色允许的操作，不额外要求审批流。

### 2.7 AI 与外部算力

AI 功能未来会通过 Nvidia 边缘设备或其他外部算力引入。树莓派不承担 AI 模型训练或重推理。

Edge 职责：

- 接收 AI 结果。
- 记录模型版本、推理结果、置信度、缺陷类型、推理耗时。
- 将 AI 结果关联到 `unit_id`、station_event、media_file。

外部设备职责：

- 图像推理。
- 高频数据复杂分析。
- 模型运行和模型管理。

## 3. 明确排除或后置

当前不作为树莓派 Edge 系统职责：

- 毫秒级或微秒级高频原始信号直接采样。
- 视频/照片长期存储。
- 大规模多工厂集中 MES。
- AI 模型训练。
- 长期大数据分析。
- 电子 SOP。
- 维护保养模块。

## 4. 推荐新增模块

| 模块 | 作用 | 优先级 |
| --- | --- | --- |
| `parameter_manager` | 参数读取、校验、快照、变更日志 | 高 |
| `file_ingestion` | MCU CSV/JSON 文件接入 | 高 |
| `feature_extractor` | 高频文件特征提取 | 高 |
| `media_manager` | 工业相机图片/视频元数据、保留策略 | 中 |
| `archive_manager` | 移动硬盘和服务器归档任务 | 中 |
| `auth_service` | 用户、角色、令牌、生物识别适配 | 中 |
| `ai_result_adapter` | 接收 Nvidia 边缘设备 AI 结果 | 低 |

## 5. 推荐新增数据表

| 表 | 说明 |
| --- | --- |
| `line_config` | 多产线配置 |
| `parameter_definition` | 参数定义、范围、单位、数据类型 |
| `parameter_snapshot` | 当前参数快照 |
| `parameter_change_log` | 参数变更日志 |
| `part_data_file` | MCU 高频采集文件元数据 |
| `signal_feature` | 高频文件提取特征 |
| `media_file` | 工业相机图片/视频元数据 |
| `archive_job` | 归档任务 |
| `archive_job_item` | 归档任务明细 |
| `user_account` | 用户账号 |
| `role_permission` | 角色权限 |
| `auth_event` | 登录、令牌、生物识别事件 |
| `operation_audit_log` | 关键操作审计 |
| `ai_inference_result` | AI 推理结果 |

## 6. 建议实施顺序

1. 参数管理与 changelog。
2. MCU 文件接入与特征提取。
3. 图片/视频元数据与 7 天保留策略。
4. 归档任务管理。
5. 权限、令牌与生物识别适配。
6. AI 结果接入预留。

## 7. 关键边界

- Edge 读取和校验 PLC 参数，不主动回写 PLC。
- 高频原始采集由 MCU 完成，Edge 处理文件结果。
- 图片/视频只做短期本地缓存和元数据追溯。
- 长期归档必须支持移动硬盘和服务器上传两个可选目标。
- 长期分析、大规模报表和 AI 重计算放在外部服务器或 Nvidia 边缘设备。
